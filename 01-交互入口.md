## 一、CLI(命令行终端)

**全称:** Command Line Interface(命令行接口)

**定义:** 通过文本命令与 AI 系统交互的终端界面,是 AI Agent 最原生、最强大的交互形态。CLI Agent 可以直接访问本地文件系统、执行 Shell 命令、循环调用工具,是实现"Agentic(智能体化)"工作流的基础。

### 1.1 为什么 CLI 是 Agent 的核心入口

CLI 之所以成为 Agent 最原生的入口,源于其四大特性:

1. **本地执行权限:** 直接读写本地文件系统、运行 Shell 命令、调用系统级 API,这是 WebUI 和 Widget 无法比拟的。
2. **多步骤规划:** 自主拆解任务,循环执行"思考 → 工具调用 → 观察 → 再思考",这是 Agent Loop 的核心机制。
3. **上下文感知:** 自动扫描项目结构、读取相关文件、维护会话状态,具备项目级智能。
4. **工具调用:** 通过 Function Calling / MCP 调用外部能力,无限扩展能力边界。

### 1.2 CLI Agent 工作流程(Agent Loop 详解)

CLI Agent 的核心是 **Agent Loop(智能体循环)**,这是一个 while 循环,LLM 自主决定何时退出:

```
用户输入任务
    ↓
[初始化阶段]
    ├── Agent 解析意图
    ├── 扫描项目上下文(目录结构、配置文件)
    └── 加载项目规则(AGENTS.md / CLAUDE.md)
    ↓
[规划阶段](可选)
    └── 制定执行步骤(Plan)
    ↓
[执行循环] ←─────────────────────┐
    ├── [1] LLM 思考:分析任务,决定下一步  │
    ├── [2] 工具调用:选择并调用 Tool       │
    │   ├── 读文件(Read)                 │
    │   ├── 写文件(Write/Edit)            │
    │   ├── 执行命令(Bash)                │
    │   ├── 搜索(Grep/Glob)               │
    │   └── MCP 工具(自定义)              │
    ├── [3] 观察结果:把工具返回加入上下文  │
    ├── [4] 判断:任务完成?                │
    │   ├── 否 → 回到 [1] 继续思考 ────────┘
    │   └── 是 → 退出循环
    ↓
[输出阶段]
    └── 输出总结报告
```

### 1.3 三级权限控制(Approval Mode)

权限控制是 CLI Agent 安全的关键机制,通常分为三档:

| 模式 | OpenAI Codex CLI | Anthropic Claude Code | 说明 | 适用场景 |
|------|------------------|----------------------|------|----------|
| **建议模式(Suggest/Chat)** | `suggest`(默认) | `--permission-mode plan` | 仅建议,所有改动需用户确认 | 咨询、规划、答疑 |
| **自动编辑(Auto-Edit)** | `auto-edit` | `--permission-mode auto-edit` | 自动编辑文件,执行命令需确认 | 日常开发、谨慎操作 |
| **完全访问(Full-Auto)** | `full-auto` | `--permission-mode full-auto` | 自主执行,无需确认(沙箱内) | 信任环境、批量任务 |

**关键概念:** Codex CLI 的 Full-Auto 模式仍然在**内核沙箱**(macOS seatbelt / Linux landlock)内执行,不会真正无限制地访问系统。

### 1.4 主流 CLI Agent 工具对比(2026)

| 工具 | 维护方 | 形态 | 核心优势 | MCP 支持 | 安装命令 |
|------|--------|------|----------|----------|----------|
| **Codex CLI** | OpenAI | 纯 CLI(Rust) | 轻量、沙箱隔离、ChatGPT 生态 | ✅ | `npm i -g @openai/codex` |
| **Claude Code** | Anthropic | 纯 CLI(Node.js) | 深度 Agentic、Skills、Subagents、Hooks | ✅ | `npm i -g @anthropic-ai/claude-code` |
| **Gemini CLI** | Google | 纯 CLI | Gemini 模型集成 | ✅ | `npm i -g @google/gemini-cli` |
| **Copilot CLI** | GitHub | 插件+CLI | GitHub 生态深度集成 | 部分 | VS Code 插件 |
| **Aider** | 社区 | 纯 CLI | 开源、Git 友好 | 部分 | `pip install aider-chat` |
| **Cline** | 社区 | VS Code 插件 | 开源、可视化 | ✅ | VS Code 插件 |

| **Skills & Plugins** | 可复用指令 + 团队工具接入 |
| **多模型支持** | 支持 GPT-5、o3、o4-mini 等推理模型 |

### 2.3 十一大终端工作流(官方定位)

| 编号 | 功能 | 命令 | 说明 |
|------|------|------|------|
| 01 | 本地仓库循环 | `codex` | 在终端单循环内探索/编辑/运行本地工具 |

| 06 | 子智能体 | - | 委派聚焦任务给 subagent |
| 07 | 网页搜索 | `codex --search "query"` | 实时网页搜索 |
| 08 | MCP 集成 | `codex mcp` | 接入本地/远程 MCP Server |

### 2.4 常用命令详解

```bash
# 启动交互式会话
codex

# 直接执行单次任务
codex "重构 utils.py 中的 parse_date 函数"

# 指定审批模式
codex --approval-mode full-auto "运行所有测试并修复失败的"
codex -a full-auto "..."

# 使用特定模型
codex --model gpt-5 "分析项目结构"
codex -m o3 "..."

# 指定推理强度(仅推理模型)
codex --model o3 -c model_reasoning_effort="high" "..."

# 恢复上次会话
codex resume

# 图像输入(视觉上下文)
codex --image screenshot.png "解释这个 UI 截图"

# 网页搜索
codex --search "查一下 React 19 的新特性"

# MCP 操作
codex mcp list                    # 列出已配置的 Server
codex mcp add <name> -- <command> # 添加 Server
codex mcp remove <name>           # 移除 Server
codex mcp serve                   # 把 Codex 自身作为 MCP Server 暴露

# 云端执行
codex cloud "在干净环境跑 CI"

# Shell 补全
codex completion bash > /etc/bash_completion.d/codex
codex completion zsh > ~/.zsh/completions/_codex
codex completion fish > ~/.config/fish/completions/codex.fish

# 权限管理(交互式)
/permissions

# 配置查看
codex config list
codex config set theme dark
```

### 2.5 ~/.codex/config.toml 完整配置示例

```toml
# ====== 模型配置 ======
model = "gpt-5"
model_reasoning_effort = "high"  # minimal / low / medium / high
model_provider = "openai"  # openai / anthropic / ollama 等

# ====== 审批与沙箱 ======
approval_mode = "suggest"  # suggest / auto-edit / full-auto
sandbox_mode = "workspace-write"  # read-only / workspace-write / danger-full-access
project_doc_max_bytes = 65536  # AGENTS.md 最大字节数

# ====== MCP Server 配置 ======
[mcp_servers.my-fs-server]
command = "python"
args = ["/path/to/server.py"]

[mcp_servers.remote-server]
url = "https://api.example.com/mcp"
env = { AUTH_TOKEN = "xxx" }

[mcp_servers.github]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-github"]
env = { GITHUB_PERSONAL_ACCESS_TOKEN = "ghp_xxx" }

# ====== 历史记录 ======
[history]
persistence = "save-all"  # save-all / none

# ====== Profiles(配置档)======
[profiles.fast]
model = "gpt-5-mini"
model_reasoning_effort = "low"

[profiles.safe]
approval_mode = "suggest"
sandbox_mode = "read-only"

[profiles.coder]
model = "gpt-5"
approval_mode = "auto-edit"
sandbox_mode = "workspace-write"

# ====== 别名 ======
[aliases]
review = "--approval-mode suggest '请审查当前 diff'"
test = "--approval-mode full-auto '运行所有测试并修复失败的'"

# 使用: codex -p fast "..." 或 codex --profile safe "..."
```

## 三、Anthropic Claude Code

### 3.1 基本信息

- **官方文档:** https://code.claude.com/docs
- **GitHub:** https://github.com/anthropics/claude-code
- **实现语言:** Node.js / TypeScript
- **配置文件:** `.claude/settings.json`(项目级) + `~/.claude/settings.json`(用户级)
- **项目规则:** `CLAUDE.md`(项目根目录)
- **MCP 配置:** `.mcp.json`(项目根目录)

### 3.2 核心特点

| 特点 | 说明 |
|------|------|
| **深度 Agentic 工作流** | 强项在于多文件、多步骤复杂任务 |
| **Skills 系统** | 可复用的技能包,遵循 AgentSkills.io 标准 |
| **Subagents / Agent Teams** | 多 Agent 协作,内置 Explore/Plan/General-purpose |
| **Hooks 钩子** | 事件驱动自动化,100% 强制执行 |
| **CLAUDE.md** | 项目级行为配置,层级覆盖 |
| **MCP 一等公民** | 原生支持 MCP Client |
| **Plan Mode** | 仅规划不执行,适合复杂任务前置规划 |
| **Cost Tracking** | 内置成本追踪 |
| **Output Format** | 支持 JSON / Stream-JSON 输出 |

### 3.3 内置 Subagent(子智能体)

Claude Code 内置三种 Subagent,运行在独立上下文窗口中:

| Subagent | 模型 | 工具权限 | 用途 | 触发场景 |
|----------|------|----------|------|----------|
| **Explore** | 继承主对话(API 上限 Opus) | 只读(禁 Write/Edit) | 文件发现、代码搜索、代码库探索 | 需要理解项目结构时 |
| **Plan** | 继承主对话 | 只读 | Plan 模式下收集上下文 | 复杂任务前的规划阶段 |
| **General-purpose** | 继承主对话 | 全部工具 | 复杂多步任务(探索 + 修改) | 默认委派任务 |

> **版本变化:** v2.1.198 起 Explore 继承主对话模型(以前固定用 Haiku)。用户/项目级同名 `Explore` subagent 会覆盖内置。

调用 Explore 时,Claude 会指定**彻底程度**:quick / medium / very thorough。

### 3.6 Hooks 系统(生命周期钩子)

| 事件 | 触发时机 | 用途 |
|------|----------|------|
| `PreToolUse` | 工具调用前 | 拦截危险操作、修改参数、记录日志 |
| `PostToolUse` | 工具调用后 | 后置处理、结果验证、格式化 |
| `Notification` | 系统通知时 | 通知转发、告警 |
| `Stop` | Agent 结束时 | 清理资源、生成报告 |
| `SubagentStop` | 子 Agent 结束时 | 子任务结果处理 |

**Hooks 配置(`.claude/settings.json`):**

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "/scripts/validate-bash.sh"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit",
        "command": "/scripts/run-linter.sh"
      }
    ]
  }
}
```

### 3.7 常用命令详解

```bash
# 启动交互式会话
claude

# 执行单次任务
claude -p "为 src/api/user.ts 添加单元测试"

# 恢复会话

# 继续上次会话
claude --continue

# 模型选择

claude --model claude-opus-4

# 权限模式
claude --permission-mode plan        # 仅规划,不执行
claude --permission-mode auto-edit   # 自动编辑,不执行命令
claude --permission-mode full-auto   # 全自动

# 成本管理
claude --cost

# 输出格式(用于脚本集成)
claude -p "..." --output-format json

# 设置
claude config set theme dark
claude config list

# 查看版本
claude --version
```

| 文件/目录 | 作用 | 范围 |
|-----------|------|------|

| `.claude/agents/` | 自定义 Subagent 定义 | 项目级 |
| `.claude/skills/` | 自定义 Skill 定义 | 项目级 |

### 3.9 CLAUDE.md 层级覆盖机制

```
~/.claude/CLAUDE.md          # 用户级(全局,所有项目生效)
项目根/CLAUDE.md              # 项目级(覆盖用户级)
项目根/src/CLAUDE.md          # 子目录级(覆盖项目级)
```

**子目录 CLAUDE.md 优先级最高**。Claude Code 进入子目录时,会合并所有层级的 CLAUDE.md 内容,后定义的规则覆盖先定义的。

## 五、Widget(嵌入式组件)

**定义:** 嵌入到其他应用中的轻量 AI UI 组件,不脱离宿主应用。

### 5.1 典型形态

| 形态 | 说明 | 典型代表 |

## 八、ASR(自动语音识别)

**全称:** Automatic Speech Recognition(自动语音识别)

**别名:** STT(Speech-to-Text,语音转文字)

**定义:** 将语音转换为文字的技术,是语音交互的输入端。

### 8.1 ASR 技术演进

| 时代 | 技术 | 代表 | 特点 |
|------|------|------|------|
| **传统时代** | HMM + GMM | Kaldi、Sphinx | 需要语言模型,效果一般 |
| **深度学习时代** | RNN/LSTM/CTC | DeepSpeech、Listen Attend Spell | 端到端,效果提升 |
| **Transformer 时代** | Transformer/Conformer | Wav2Vec 2.0、Hubert | 自监督预训练 |
| **大模型时代** | 多模态大模型 | Whisper、Gemini | 多语言、鲁棒性强 |

### 8.2 OpenAI Whisper

**官网:** https://openai.com/research/whisper
**论文:** "Robust Speech Recognition via Large-Scale Weak Supervision"(2022)
**开源仓库:** https://github.com/openai/whisper
**HuggingFace:** https://huggingface.co/openai/whisper-large-v3

**核心特点:**

| 特点 | 说明 |
|------|------|
| **多语言支持** | 99 种语言 |
| **开源可商用** | MIT License |
| **多任务** | 转录 + 翻译 + 语言识别 + VAD |
| **鲁棒性强** | 训练于 68 万小时多语言数据 |
| **变体丰富** | tiny/base/small/medium/large-v1/v2/v3 |

**Whisper 模型变体对比:**

| 模型 | 参数量 | 显存需求 | 速度 | 相对准确度 |
|------|--------|----------|------|-----------|
| tiny | 39M | ~1GB | 最快 | ★ |
| base | 74M | ~1GB | 很快 | ★★ |
| small | 244M | ~2GB | 快 | ★★★ |
| medium | 769M | ~5GB | 中 | ★★★★ |
| large-v1 | 1550M | ~10GB | 慢 | ★★★★☆ |
| large-v2 | 1550M | ~10GB | 慢 | ★★★★★ |
| large-v3 | 1550M | ~10GB | 慢 | ★★★★★★(最新) |

**Whisper 使用示例:**

```python
import whisper

model = whisper.load_model("large-v3")
result = model.transcribe("audio.mp3", language="zh")
print(result["text"])
print(result["segments"])  # 带时间戳的分段
```

## 十、流式输出(Streaming)

**定义:** 模型边生成边输出,用户可以实时看到生成过程,而不是等全部完成后才显示。

### 10.1 流式输出的实现方式

| 方式 | 全称 | 说明 | 适用场景 |
|------|------|------|----------|
| **SSE** | Server-Sent Events | HTTP 长连接,服务端推送 | Web 应用、LLM API |
| **WebSocket** | WebSocket | 全双工通信 | 实时交互、双向通信 |
| **HTTP Chunked** | HTTP 分块传输 | 简单分块 | 简单场景 |
| **gRPC Stream** | gRPC 流式 | 高性能 RPC | 微服务 |

### 10.2 SSE 详解(Server-Sent Events)

SSE 是 LLM API 流式输出最常用的协议,基于 HTTP 长连接:

```
GET /v1/chat/completions HTTP/1.1
Content-Type: application/json
stream: true

HTTP/1.1 200 OK
Content-Type: text/event-stream

data: {"id":"...","choices":[{"delta":{"content":"你"}}]}

data: {"id":"...","choices":[{"delta":{"content":"好"}}]}

data: [DONE]
```

### 10.3 流式输出的优势

| 优势 | 说明 |
|------|------|
| **用户体验好** | 可以看到实时进度,减少等待感 |
| **可中断** | 用户可以中途停止 |
| **节省时间** | 不需要等全部生成完 |
| **首 Token 快** | TTFT(Time To First Token)低 |
| **适合长输出** | 长文本生成不必等待 |

## 十四、典型应用场景

### 14.1 软件开发场景

```
开发者
   ↓
[Cursor / Windsurf / Trae] ← IDE 内 Copilot
   ↓ 需要复杂任务
[Claude Code / Codex CLI] ← CLI Agent
   ↓ 需要工具扩展
[MCP Server] ← 接入外部能力
```

### 14.2 客服场景

```
客户
   ↓
[网站 Widget] ← 浮窗入口
   ↓ 复杂问题
[Chatbot 平台] ← Dify / Coze
   ↓ 需要业务系统
[API + 工具调用] ← 集成业务系统
```

### 14.3 语音助手场景

```
用户语音
   ↓
[ASR: Whisper] ← 语音转文字
   ↓
[LLM 对话] ← 理解 + 生成
   ↓
[TTS: OpenAI TTS] ← 文字转语音
   ↓
用户听到回复
```
